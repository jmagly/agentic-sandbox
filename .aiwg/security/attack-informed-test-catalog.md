# Attack-Informed Agent Abuse Regression Catalog

Date: 2026-06-22

Scope: local-first Agentic Sandbox deployments through `v2026.6.28`.
This catalog maps the existing STRIDE threat models to ATT&CK-informed
agent-abuse regression scenarios. It is a test planning and evidence index, not
a formal ATT&CK emulation plan or penetration-test report.

## Status Terms

| Status | Meaning |
| --- | --- |
| Automated | A repository test or check already exercises the scenario. |
| Manual | A repeatable manual verification path exists, but it is not automated. |
| Planned | The scenario is required, but implementation or harness work is still open. |
| Not applicable | The tactic is intentionally out of scope for the current runtime/profile. |

## Review Notes

- ATT&CK mappings are approximate because this system models agent sandbox abuse
  rather than enterprise endpoint intrusion. The tactic labels are used to make
  regression intent searchable and to ensure credential access, lateral
  movement, privilege escalation, exfiltration, persistence, defense evasion,
  discovery, and impact are represented.
- Planned scenarios should graduate to automated status only when a test records
  negative evidence with fake sentinel values and can run without real
  credentials or external provider access.
