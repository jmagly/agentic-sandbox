# VM Loadout Specification

This document defines the required components for agent VMs in the agentic-sandbox system.

## Core Components (Required)

These must be present and functioning for the VM to operate correctly.

### 1. Agent Client Binary

| Item | Path | Source |
|------|------|--------|
| Binary | `/opt/agentic-sandbox/bin/agent-client` | `agent-rs/target/release/agent-client` |
| Service | `/etc/systemd/system/agent-client.service` | `agent-rs/systemd/agent-client.service` |
| Config | `/etc/agentic-sandbox/agent.env` | Generated by cloud-init at provision |

**Verification:**
```bash
systemctl status agent-client
# Should show: active (running)
```

### 2. Agent Environment File

Generated at provision time with unique credentials:
```bash
# /etc/agentic-sandbox/agent.env
AGENT_ID=agent-01
AGENT_SECRET=<256-bit-hex>
MANAGEMENT_SERVER=host.internal:8120
```

### 3. Runtime Directories

| Directory | Owner | Purpose |
|-----------|-------|---------|
| `/opt/agentic-sandbox/bin/` | root | Agent binaries |
| `/var/run/agentic-sandbox/` | agent | Runtime state (PID files, markers) |
| `/etc/agentic-sandbox/` | root | Configuration |
| `/var/log/agentic-setup.log` | root | Provisioning log |

### 4. Network Configuration

- **host.internal** must resolve to the host gateway (192.168.122.1)
- Port 8120 (gRPC) must be reachable from VM
- Added via cloud-init: `/etc/hosts` entry

### Container Runtime

**Rootless Docker** - No privilege escalation via docker socket.

```bash
# Verification
docker --version
docker run --rm hello-world
```

Key settings:
- User NOT in docker group (security)
- Docker daemon runs as agent user
- Socket at `/run/user/1000/docker.sock`
- Enabled via `loginctl enable-linger agent`

## Shell Configuration

### ~/.bashrc Additions

```bash
# Rootless Docker
export XDG_RUNTIME_DIR="/run/user/$(id -u)"
export DOCKER_HOST="unix://${XDG_RUNTIME_DIR}/docker.sock"

# Tool paths
export PATH="$HOME/.local/bin:$PATH"
export PATH="$HOME/.local/share/fnm:$PATH"
export PNPM_HOME="$HOME/.local/share/pnpm"
export BUN_INSTALL="$HOME/.bun"
export GOPATH="$HOME/.local/go"
export PATH="/usr/local/go/bin:$GOPATH/bin:$PATH"
export PATH="$BUN_INSTALL/bin:$PATH"

# Tool initialization
eval "$(fnm env --use-on-cd 2>/dev/null)" || true
source "$HOME/.cargo/env" 2>/dev/null || true
eval "$(mise activate bash 2>/dev/null)" || true
eval "$(direnv hook bash 2>/dev/null)" || true

# Aliases
alias bat='batcat'
alias fd='fdfind'
```

## Deployment Order

1. **Base image** - Ubuntu 24.04 cloud image
2. **cloud-init** - Injects user-data with write_files and runcmds
3. **apt packages** - System packages installed
4. **agent-client** - Binary + service deployed
5. **Rootless Docker** - Container runtime configured
6. **Go runtime** - System-level install
7. **User tools** - uv, fnm, rust, etc. (runs as agent)
8. **Shell config** - bashrc additions
9. **Ready marker** - `/var/run/agentic-setup-complete`

## Troubleshooting

### Agent not connecting

```bash
# Check service
journalctl -u agent-client -f

# Check network
curl -v http://host.internal:8120

# Check credentials
cat /etc/agentic-sandbox/agent.env
```

### Tools not in PATH

```bash
# Reload shell
source ~/.bashrc

# Or start new session
exec bash -l
```

### Docker not working

```bash
# Check rootless daemon
systemctl --user status docker

# Check socket
ls -la $XDG_RUNTIME_DIR/docker.sock

# Restart if needed
systemctl --user restart docker
```
