# Start Here

Use this section when you are deciding how to approach Agentic Sandbox for the
first time.

## Choose A Route

| Route | Best for | Start |
| --- | --- | --- |
| **Fastest validation** | Start the management server, create a container runtime, attach a live terminal, and prove the dashboard loop works. | [Getting Started](../getting-started.md) |
| **Runtime choice** | Compare isolation, startup, storage, and recovery tradeoffs before provisioning anything durable. | [Runtime Map](../runtimes/overview.md) |
| **Mental model** | Read the three surfaces, management server, agent client, storage, lifecycle, and AIWG executor story in order. | [Architecture Map](../architecture/overview.md) |
| **Production path** | Move from a local run to deployment, monitoring, reliability, and troubleshooting workflows. | [Operations Map](../operations/overview.md) |

## Reading Order

1. [Getting Started](../getting-started.md) - local install, dashboard path,
   container runtime, VM runtime, first task.
2. [Core Concepts](../concepts.md) - naming, task lifecycle, three surfaces,
   and synchronization terms.
3. [Platform Support](../platform-support.md) - supported hosts, guest images,
   hypervisors, and runtime status.
4. [Glossary](../glossary.md) - term-level definitions used across the docs.
